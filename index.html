Сервисы транскрибации (ASR)

📝 Ключевые различия

  • Тарификация: Deepgram — посекундная (экономия 15–30%), OpenAI — поминутная, Yandex — по 15-сек отрезкам (округление вверх)
  • Диаризация (спикер 1 / спикер 2): у Deepgram отдельно ~$0.001–0.002/мин, у Whisper — нет нативной, нужен pyannote.audio
  • Русский язык: Yandex SpeechKit — лучший (95–97%), Whisper — хороший (~93%), Deepgram — ограниченная поддержка RU
  • Self-hosted Whisper: GPU ~$0.5/час (RunPod), обрабатывает ~60× realtime → ~$0.001/мин, но нужен DevOps
  • GPT-4o-mini Transcribe: $0.003/мин — самый дешёвый облачный вариант с отличным качеством, но только batch
  • Streaming: Deepgram (<300ms) и Yandex sync — единственные для real-time подсказок агентам

LLM для анализа звонков

Расчёт: ~4 000 input + ~800 output токенов на звонок (промпт + чек-лист + транскрипт + оценка)

📝 Как снизить стоимость LLM на 60–80%

  • Prompt caching (−90%): Claude и GPT кэшируют system prompt. Ваш чек-лист отправляется каждый раз → идеальный кандидат для кэша
  • Batch API (−50%): OpenAI даёт 50% скидку на async запросы с доставкой за 24 ч — идеально для пост-анализа звонков
  • Model routing: простые чек-листы (да/нет) → GPT-4o Mini, сложный анализ → Claude Haiku/Sonnet. Экономия 60–80%
  • Output ≥ 3× input по цене — просите модель возвращать JSON-схему, а не свободный текст. Меньше токенов = меньше денег
  • GigaChat: данные хранятся в РФ (152-ФЗ), дешевле Claude, но хуже следует сложным инструкциям для структурированного скоринга
  • DeepSeek V3.2: $0.28/$0.42 — неожиданно хорошее качество за минимальные деньги, но серверы в Китае

🎯 Итог: ASR = 60–80% затрат, LLM — дёшев

Главный инсайт: транскрибация стоит в разы дороже LLM-анализа. При 100 звонках/день по 5 минут: ASR обойдётся в $5–80, а LLM — в $1–5. Оптимизируйте ASR в первую очередь (self-hosted Whisper, batch mode, посекундная тарификация). Для LLM используйте prompt caching + model routing — и стоимость анализа станет почти незаметной.
